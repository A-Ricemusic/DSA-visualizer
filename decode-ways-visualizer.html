A number can become a letter in one or two digits. The trick is not listing every message—it is counting how many valid paths reach each position.

The whole idea

At each digit, look back one step and two steps.

Why 3? The final 6 adds 2 paths, and the valid pair 26 adds 1 more.

01 · First principles

Only three rules

Every decision in the algorithm comes from these rules. If you can check one digit and two digits, you can solve the problem.

1

One digit: 1–9

A single non-zero digit is a letter. 7 → G. A single 0 is never a letter.

2

Two digits: 10–26

A pair is a letter only when its number falls in this range. 12 → L, but 27 is too large.

3

Add both valid paths

If both choices work, add the path counts that existed before them. That repeated work is exactly what DP saves.

03 · Why DP?

Save the answer to every smaller prefix.

When a choice lands at position i, we do not recount everything before it. We reuse the number already stored there.

dp[i] means: “How many valid ways decode the first i characters?”
Think of paths climbing stairs
1empty
12
222
3226

A valid 1-digit letter is a one-step jump. A valid 2-digit letter is a two-step jump. Count the ways to land on the final stair.

04 · The common trap

Zero needs a partner

0 has no letter by itself. It only works inside 10 or 20. Treat this as a rule check, not a special patch.

“10”
10→ J

One valid path. The single 0 fails, but the pair 10 works.

“101”
101→ JA

Still one path. You cannot split it as 1 | 0 | 1.

“06”
0 ✕06 ✕

Zero paths. Leading zero is invalid, and 06 is not a valid pair.

05 · Worked examples

2|2|6 → BBF, 22|6 → VF, and 2|26 → BZ. The pair 26 is just inside the limit.
s = “27”1 way
2 | 7 → BG works, but 27 is larger than 26. A string without zeros can still have only one decoding.
s = “11106”2 ways
1|1|10|6 → AAJF and 11|10|6 → KJF. The zero forces the preceding 1 to join it.
06 · Translate to code

The Python solution

The code is short because the table already captured all the reasoning. Each loop performs exactly the two rule checks from above.

solution.pyO(n) time · O(n) space
class Solution:
    def numDecodings(self, s: str) -> int:
        # dp[i] = ways to decode the first i chars
        dp = [0] * (len(s) + 1)
        dp[0] = 1  # one way to decode nothing

        for i in range(1, len(s) + 1):
            # Take one digit: 1 through 9
            if s[i - 1] != "0":
                dp[i] += dp[i - 1]

            # Take two digits: 10 through 26
            if i >= 2 and "10" <= s[i - 2:i] <= "26":
                dp[i] += dp[i - 2]

        return dp[len(s)]
1

Give the empty prefix one way

dp[0] = 1 is the starting path. It lets the first valid letter contribute one solution.

2

Try the one-digit door

If the current character is not zero, every path at dp[i-1] can extend through it.

3

Try the two-digit door

If the pair is 10–26, every path at dp[i-2] can extend through that pair.

4

Return the last cell

It represents the entire string, so it contains the total number of valid decodings.
